# Report exports

Reports are stored as validated TipTap JSON version `1` and exported from authenticated server routes after project and report ownership checks. The canonical document rejects unsupported nodes/marks, unsafe links, oversized serialized content, excessive node counts, deep nesting, and excessive text.

Supported export formatting: paragraphs, level 1-3 headings, bold, italic, inline code, absolute HTTP/HTTPS links, bullet and ordered lists with indentation, block quotes, code blocks, hard breaks, and basic tables with header/data cells. HTML output is generated by escaping text and rendering only validated nodes/marks into a standalone UTF-8 document with print-friendly CSS. Markdown escapes syntax characters where needed and includes table separators. PDF uses `@react-pdf/renderer` in a Node.js route and recursively renders supported nodes with metadata, safe wrapping, links, tables, code blocks, lists, and page numbers.

Limitations: unsupported TipTap extensions are rejected by validation rather than exported; advanced editor table controls such as merged cells are not supported; PDF output is readable structured report output, not pixel-perfect browser-editor reproduction.
